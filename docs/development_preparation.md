## Step 1: プロダクトの目的・ビジョン

### 1. プロダクトの目的

- **生成AIエージェントによるスクラム開発チームを提供するチャットアプリ**  
- ユーザーはチャットでアプリのアイデアを伝えると、AIエージェント（スクラムチーム）が開発計画から実装までを支援。  
- **既存のローコード/ノーコードツールでは不十分なカスタマイズ**を、ユーザー自身が容易に行える点が特徴。

### 2. ターゲットユーザー

- **自分でアプリをカスタマイズしたいユーザー**  
  - アプリ開発スキルがあまりない起業家、個人開発者  
  - プロトタイプを素早く作って検証したい方  
- **開発初心者から中級者**を想定。AIエージェントがスクラム的な進め方をナビゲート。

### 3. 将来のビジョン (2〜3年後)

- AIエージェントがデザインやビジネス面まで提案できるようになり、**起業や新規事業立ち上げのハードルを劇的に下げるプラットフォーム**を目指す。  
- 多様な外部APIとの連携も自動化し、**チャットするだけで複雑なアプリを構築できる**未来へ。

## Step 2: スコープの明確化とリリースゴールの設定

### 1. MVP で実装する機能

1. **Chat UI**  
   - ユーザーとエージェントがテキストベースでやり取りできる最小限のチャット機能  
2. **エージェントチームによるスクラム支援**  
   - PBI（Product Backlog Item）の作成、タスク分解などを行うエージェント  
3. **開発者エージェントによるコーディング**  
   - チャット上の指示に応じてコーディングを自動生成・提案  
4. **HITL (Human in the Loop) による人とAIの開発協調**  
   - エージェントの提案にユーザーが修正指示を出して、協働で開発を進める

> **後回しにする機能**  
> - ユーザー管理（本格的な認証やプロフィール管理など）

### 2. リリース段階のイメージ

- **α版**: 2025年2月10日までに完成  
  - MVP 機能を検証し、ユーザーフィードバックを得て今後の方針を決定  
- **β版 / 正式版**: α版の結果を踏まえてスケジュールを再策定

### 3. α版で実現したいユーザー体験

- **一人でもエージェントと協調してアジャイル開発を進められる**  
- **アイデアを素早く形にし、必要に応じてカスタマイズ**が可能

## Step 3: プロダクトバックログ案の作成

### 1. バックログ管理ツール
- **[GitHub Project](https://github.com/users/teeppp/projects/5)**  
  - PBIやタスクを登録し、To Do / In Progress / Done で進捗管理

### 2. 主なユーザーストーリー & タスク例

| ユーザーストーリー/タスク名                                         | 内容                                                                                               | 受け入れ条件 (例)                                                                                              |
|:--------------------------------------------------------------------|:----------------------------------------------------------------------------------------------------|:----------------------------------------------------------------------------------------------------------------|
| **US-1: 「アプリを作りたい」と入力 → PBIの自動提案**                 | チャットUIで要望を伝えるとエージェントが10個のPBIを生成し、GitHub Projectに登録する                     | 1. PBI案を自動生成<br>2. GitHubへのAPI連携<br>3. ユーザーにPBI一覧をチャット表示                                |
| **US-2: PBIをTaskに分解 (開発エージェント)**                         | 作成されたPBIを細分化し、コーディング可能なレベルにする                                             | 1. TaskがPBIとリンク<br>2. タスク内容が具体的<br>3. GitHub Projectでタスク管理                                  |
| **US-3: タスクからコーディングを実装 (AIコーディングアシスタント)**   | 生成したタスクをAIがコードに落とし込み、リポジトリへプッシュ                                        | 1. ユーザーの指示→AIがコード提案<br>2. CI/CDでビルド/テスト                                                     |
| **US-4: PBIの受け入れ条件チェック (AIエージェントフレームワーク)**   | PBIの完了判定に際し、AIエージェントがテストシナリオ等を自動生成・実行                                  | 1. エージェントが受け入れテスト<br>2. 問題なければDone                                                            |

### 3. 優先度付けの方針

- **開発難易度**を軸に、LLMエージェントPOがユーザーと相談しながら決定  
  - まずはコア機能（ChatUIとPBI登録）が優先度【高】  
  - 受け入れテストの自動化は【中】、ユーザー管理機能は【低】 など

## Step 4: スクラムイベント計画

### 1. スプリントの期間
- **1週間**  
  - 細かいサイクルで素早いフィードバックと改善

### 2. スクラムイベントの方法・ツール
- **メインツール**: ChatGPT  
  - スプリントプランニング、デイリースクラム、レトロスペクティブ等を ChatGPT または他のチャットツールで  
- **チーム内コミュニケーション**: GitHub Project のIssueやPRと併用

### 3. α版までのスプリント数
- **4スプリント**（1スプリント = 1週間 × 4回 = 約1ヶ月）  
  - 2月10日 α版リリースを目指す

## Step 5: スプリント0（キックオフ）の準備

### 1. スプリント0の目的
- **Google Cloud + Flutter の開発環境**を整備し、サーバレス構成での“Hello World”を確認  
- プロジェクトの雛形づくり (GitHub Project設定、CI/CD初期設定 など)

### 2. スプリント0で実施したいタスク（例）

1. **GCP プロジェクト作成 / 初期設定**  
   - サーバレスに必要なサービス (Cloud Run / Cloud Functions / Firestoreなど) を有効化  
2. **Flutter “Hello World”**  
   - ローカルで Flutter プロジェクト作成 → ビルド確認  
3. **サーバレス構成での “Hello World”**  
   - Cloud Run でPythonアプリ起動 → Flutterからリクエスト → “Hello World”レスポンス  
4. **GitHub Project の初期設定**  
   - タスク登録 → To Do / In Progress / Done の流れを試す  
5. **LLMエージェント連携の下準備**  
   - APIキーの管理・アクセス権限などをざっくり把握

### 3. キックオフ時に確認すべき事項
- コーディング規約やLint設定は都度決める  
- GCPの無料枠や課金設定の確認  
- スプリント開始/終了のタイミングを1週間区切りで決定

## Step 6: スプリント1の計画

### 1. スプリントゴール
- **Flutter画面でFirebase経由→Cloud Run→Vertex AI (LLM) と会話ができる**ところを目指す  
- ユーザーが入力したテキストをサーバレスAPIに送り、LLM応答を取得・表示

### 2. スプリント1で取り組みたいタスク

| タスクID | タスク名                                                | 内容                                                                                                      |
|:--------:|:-------------------------------------------------------|:----------------------------------------------------------------------------------------------------------|
| **T-1**  | Firebaseプロジェクト設定                               | - Firebaseでプロジェクト作成、認証/Firestore等のサービス有効化                                            |
| **T-2**  | FlutterアプリにFirebase SDK導入                        | - FlutterプロジェクトへAuthやFirestoreのSDKを追加し、接続確認                                            |
| **T-3**  | Cloud Run (Python) でLLM呼び出しAPI作成                | - Python (FastAPI等) でREST API<br>- Vertex AI API (Gemini) へのリクエスト実装                             |
| **T-4**  | FlutterからCloud Run APIを呼び出す                    | - FlutterのHTTPクライアント実装 (Dio/httpパッケージ)<br>- 必要に応じてFirebase Authトークンを添付           |
| **T-5**  | チャットUI画面実装                                    | - メッセージ一覧/入力欄/送信ボタンなど、シンプルなチャット画面                                             |
| **T-6**  | 動作検証 & Debug                                      | - 実機でLLMの応答が返るか確認<br>- 必要に応じてログ出力やエラーハンドリング                                |

### 3. 見積もり & 受け入れ基準
- **見積もり**: タスク数に応じてストーリーポイントを割り振り、1週間で対応できる範囲を調整  
- **受け入れ基準**  
  - 「Flutterアプリを起動し、テキストを入力 → Cloud Run経由でLLM応答が返り、画面表示される」ことが確認できれば Done
。
